## 1. Executive Summary
This addendum formalises four capability enhancements derived from a structured competitive intelligence analysis of Agent-Shield v9.0 (April 2026). The analysis identified specific modules in the open-source project that address genuine enterprise security gaps — gaps that Sphinx either does not yet cover or covers less precisely than the competitive benchmark.

These four enhancements are classified as Roadmap v1 additions and extend the existing Sphinx PRD v2.0 (30 sprints, 8 phases) with a new Phase 9 comprising Sprints 31–36 (Weeks 61–72). They do not replace any existing sprint work; they extend the roadmap forward.

## 2. Strategic Rationale
The Agent-Shield competitive analysis surfaced four tactical gaps in Sphinx relative to an open-source reference implementation. Closing these gaps has three strategic effects:

- Detection depth: Sphinx moves from pattern-matching to embedding-based semantic analysis for RAG injection — a materially harder-to-bypass detection paradigm.

- Evidence chain: Canary tokens and model fingerprinting create forensic artefacts that enterprise buyers (BFSI, healthcare) require for incident response and regulatory audit.

- Compliance packaging: An OWASP LLM Top 10 v2025 coverage matrix converts Sphinx's existing capabilities into a verifiable compliance artefact — reducing sales friction in regulated-sector deals.

### 2.1 Why These Four, Why Now
Agent-Shield's IPIA module and canary token implementation are the two features most cited in Q1 2026 security analyst reports as gaps in enterprise LLM firewalls. Model fingerprinting is directly triggered by the March 2026 Meta rogue AI agent incident, which demonstrated model-swap as an attack vector in shared inference infrastructure. The OWASP v2025 matrix is a procurement requirement in at least three known deals in the TrustFabric pipeline.

### 2.2 Architectural Fit
All four modules integrate cleanly into existing Sphinx interception layers. E15 (IPIA) attaches to the pre-context-injection stage already present in the Sphinx gateway. E16 (Canary) attaches to system prompt assembly. E17 (Fingerprinting) attaches to output scanning. E18 (OWASP matrix) is a reporting module over existing capability tags. No new gateway components are required; these are additive modules on existing intercept points.

## 3. Enhancement Modules (E15–E18)
### 3.1 E15 — Indirect Prompt Injection Attack (IPIA) Pipeline
Accelerated RAG Vulnerability Scanner with Joint-Context Embedding Detection

|                                                      |
|------------------------------------------------------|
| **E15 \[P0\] IPIA Joint-Context Embedding Pipeline** |

| **Field**             | **Detail**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
|-----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Problem Statement** | Sphinx's current RAG vulnerability scanner uses pattern matching against known injection templates. This approach fails on novel or paraphrased injection payloads embedded in retrieved chunks. A retrieved chunk can carry instructions that are syntactically benign (no trigger words) but semantically redirect the agent — a class of attack that pattern matching structurally cannot catch.                                                                                                                                                                                                                                                                                                       |
| **Solution Approach** | Implement a joint-context embedding classifier as the primary IPIA detection layer. The approach: (1) encode the retrieved chunk and user query as a joint embedding pair; (2) compute the semantic deviation between the chunk's content and the expected retrieval intent for that query; (3) flag chunks that instruct the agent rather than inform it. A lightweight cosine similarity threshold (configurable, default 0.5) determines the injection boundary. Agent-Shield's IPIA module validates this approach with F1=100% on HackAPrompt/TensorTrust benchmarks. Sphinx adopts the same architecture with FastAPI embedding service backed by SentenceTransformers (all-MiniLM-L6-v2, on-prem). |
| **OWASP Coverage**    | LLM08-2025 (Vector and Embedding Weaknesses), OWASP Agentic AI AA7 (Data/Context Manipulation via External Sources)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Competitive Gap**   | No enterprise LLM firewall has shipped joint-context embedding as the primary RAG injection detection layer. Lakera Guard uses pattern + model classification. Palo Alto Prisma AIRS uses heuristics. First-mover window estimated at 2–3 quarters.                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Key Deliverables**  | Joint-context embedding service (SentenceTransformers, FastAPI, on-prem); Batch RAG scan API — scan all chunks before context injection; Per-chunk confidence score (0.0–1.0) with reason output; IPIA threat event emitted to TrustDetect; Configurable threshold and embedding backend (pluggable: MiniLM, OpenAI-compatible local endpoints); Admin dashboard widget: IPIA detection rate, top blocked chunk categories                                                                                                                                                                                                                                                                                |
| **Effort Estimate**   | 2 sprints (Sprints 31–32)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Business Value**    | Blocks a class of attack that pattern matching cannot catch. Directly addresses OWASP LLM08-2025 and Agentic AA7. Converts Sphinx from heuristic-grade to semantic-grade RAG protection — a material differentiation in enterprise RFPs. Required for BFSI RAG deployments where context poisoning is the primary attack vector.                                                                                                                                                                                                                                                                                                                                                                          |

### 3.2 E16 — Canary Token Module
System Prompt Leakage Detection via Session-Scoped Synthetic Markers

|                                                               |
|---------------------------------------------------------------|
| **E16 \[P0\] Canary Token — System Prompt Leakage Detection** |

| **Field**             | **Detail**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|-----------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Problem Statement** | System prompt extraction — where an attacker induces an agent to repeat its system prompt verbatim — is OWASP LLM07-2025 (System Prompt Leakage) and among the most common attacks in enterprise deployments. Sphinx's existing system prompt guard uses pattern matching on input requests. It does not monitor agent outputs for leakage that has already occurred. There is no forensic mechanism to confirm whether a system prompt has been leaked in a given session.                                                                                                                                                                                                                 |
| **Solution Approach** | Inject a unique HMAC-signed canary string (UUID v4 + session ID + HMAC-SHA256, truncated to 12 chars) into the system prompt preamble at session creation. The canary is invisible to normal agent operation. The Sphinx output scanner monitors all agent responses for canary re-appearance. If detected, a CRITICAL severity threat event is emitted with session ID, turn number, matched canary, and suspected extraction vector. Canaries rotate per session and are never logged in audit trails (only stored in an in-memory session store TTL=session duration). Architecture is derived from Agent-Shield's CanaryTokens module, adapted for Sphinx's gateway interception model. |
| **OWASP Coverage**    | LLM07-2025 (System Prompt Leakage)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Competitive Gap**   | No competitor has a session-scoped canary token system as part of their LLM firewall. Lakera and Palo Alto rely entirely on input-side detection. Canary tokens provide output-side forensic confirmation — a qualitatively different evidence class that enterprise SOC teams require.                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Key Deliverables**  | CanaryTokenGenerator: HMAC-SHA256 signed, session-scoped, TTL-managed; System prompt injection hook at Sphinx session initialisation; Output scanner regex: canary pattern match per response turn; CRITICAL threat event → TrustDetect on leakage detection; Canary audit log: session ID, detection timestamp, turn index, extraction confidence (never logs the canary string itself); Admin toggle: enable/disable per policy; Compliance report annotation: LLM07-2025 coverage confirmed via canary monitoring                                                                                                                                                                        |
| **Effort Estimate**   | 1 sprint (Sprint 33)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Business Value**    | Provides the only output-side forensic confirmation of system prompt leakage in the Sphinx stack. Required for regulated industries where breach confirmation (not just prevention) is a compliance obligation. Closes a gap that Agent-Shield has shipped and Sphinx does not yet have. Fast to build (1 sprint), high perceived value.                                                                                                                                                                                                                                                                                                                                                    |

### 3.3 E17 — Model Fingerprinting & Supply Chain Integrity
Stylistic Response Analysis for Model Swap and Inference Endpoint Compromise Detection

|                                                              |
|--------------------------------------------------------------|
| **E17 \[P1\] Model Fingerprinting & Supply Chain Integrity** |

| **Field**             | **Detail**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
|-----------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Problem Statement** | In shared inference infrastructure (on-prem GPU clusters, multi-tenant cloud endpoints), an attacker who compromises the inference endpoint can silently substitute a different or poisoned model. This attack is undetectable by input-side scanning — the gateway passes legitimate requests and receives responses that appear normal. The March 2026 Meta rogue AI agent incident demonstrated model identity as an exploitable supply chain vector. No existing Sphinx module detects model swap.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Solution Approach** | Build a stylistic response fingerprinting engine that establishes a baseline profile for the expected inference model at Sphinx deployment time. The engine extracts 16+ stylometric features from model responses: token entropy distribution, punctuation density, average sentence length, paragraph count distribution, refusal phrasing pattern frequency, hedging language frequency (probably/may/might), bullet list usage rate, code block frequency, numbered list frequency, citation pattern presence, question-ending frequency, response length distribution, capitalisation patterns, conjunctive adverb usage, passive voice frequency, and negation density. A baseline profile is computed over 50–100 warm-up responses at deployment. In production, each response is scored against the baseline; a z-score deviation above a configurable threshold (default: 2.5σ) triggers a HIGH severity threat event with the feature vector delta. Supply chain swap detection: automatic alert if consecutive deviation scores sustain above threshold for 5+ responses. |
| **OWASP Coverage**    | LLM03-2025 (Model/Algorithm Confusion), OWASP Agentic AI AA5 (Supply Chain Compromise)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Competitive Gap**   | Agent-Shield v9.0 ships ModelFingerprinter with 16 stylistic features and SupplyChainDetector with configurable consecutive-response thresholds. No other enterprise LLM firewall has this capability. First-mover window is genuine — Prisma AIRS, Lakera, and Radware Agentic do not have model fingerprinting as of Q1 2026.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Key Deliverables**  | StylemetricFeatureExtractor: 16 feature extractors, per-response scoring; BaselineProfiler: 50-response warm-up, JSON profile export/import (for reproducible deployments); DeviationScorer: z-score per feature, configurable alert threshold (default 2.5σ); SupplyChainMonitor: consecutive-response alerting (configurable window, default 5); HIGH severity threat event → TrustDetect on model swap detection; Inference endpoint health dashboard: rolling baseline comparison, feature drift chart; Admin API: reset baseline, export profile, configure thresholds                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Effort Estimate**   | 2 sprints (Sprints 34–35)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Business Value**    | Addresses an attack vector (model swap in shared inference) that has no other mitigation in the Sphinx stack. Directly triggered by the March 2026 Meta rogue AI agent incident. Required for on-prem enterprise deployments with shared GPU infrastructure. Differentiates Sphinx from all named competitors.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |

### 3.4 E18 — OWASP LLM Top 10 v2025 Compliance Matrix
Per-Category Coverage Dashboard and Exportable Compliance Report

|                                                                  |
|------------------------------------------------------------------|
| **E18 \[P1\] OWASP LLM Top 10 v2025 Compliance Coverage Matrix** |

| **Field**             | **Detail**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
|-----------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Problem Statement** | Enterprise buyers — particularly in BFSI, healthcare, and public sector — require documented evidence of coverage against named security standards. OWASP LLM Top 10 (2025 edition, published December 2025) is the most commonly cited standard in LLM security RFPs. Sphinx has capabilities that cover most of LLM01–LLM10, but this coverage is not mapped, documented, or exportable in a buyer-facing format. Sales teams currently produce this mapping manually per deal, inconsistently.                                                                                                                                                                                                                                                                                                                                                                             |
| **Solution Approach** | Build an OWASP LLM Top 10 v2025 coverage matrix as a native Sphinx compliance module. Each Sphinx capability is tagged with the OWASP categories it addresses (one-to-many). The matrix computes per-category coverage scores (0–100%) based on which tagged capabilities are active in the current deployment configuration. The module produces: (1) a per-category coverage dashboard (UI widget); (2) a gap analysis listing uncovered requirements per category; (3) remediation guidance for gaps; and (4) an exportable compliance report in PDF and JSON. The coverage scoring methodology follows Agent-Shield's OWASPCoverageMatrix design but is adapted to Sphinx's capability taxonomy. The module is configuration-aware: if an admin has disabled a capability (e.g., IPIA scanner off), the relevant OWASP categories automatically reflect reduced coverage. |
| **OWASP Coverage**    | LLM01-2025 (Prompt Injection) through LLM10-2025 (Unbounded Consumption) — full Top 10 coverage mapping                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Competitive Gap**   | Lakera Guard and Palo Alto Prisma AIRS both have compliance reporting, but neither produces a per-category OWASP LLM Top 10 v2025 coverage score with gap analysis. Agent-Shield ships OWASPCoverageMatrix (v6.0) as an open-source module. Sphinx adding this as an enterprise-grade exportable artefact — with configuration-aware scoring — closes a procurement blocker in at least 3 known pipeline deals.                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Key Deliverables**  | OWASP LLM Top 10 v2025 capability tag schema (tags applied to all existing and new Sphinx modules); Per-category coverage score engine (0–100%, configuration-aware); Gap analysis: uncovered requirements per LLM01–LLM10, ranked by coverage deficit; Remediation guidance: actionable steps for each gap, linked to Sphinx configuration options; Compliance dashboard widget: radar chart (per-category scores), overall Shield Score (0–100); PDF compliance report export (branded, customer-facing); JSON export for programmatic integration (GRC platforms, SIEM); Automated re-scoring on configuration change                                                                                                                                                                                                                                                      |
| **Effort Estimate**   | 1 sprint (Sprint 36)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Business Value**    | Converts a sales blocker into a sales accelerator. An exportable OWASP LLM Top 10 v2025 compliance report is a procurement requirement in at least 3 active TrustFabric pipeline deals. Requires minimal new engineering (primarily tagging existing capabilities and building the scoring/reporting layer). High ROI: 1 sprint, immediate commercial impact.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |

## 7. Out of Scope — Roadmap v1
